## [과제 02 - 1] 📖 HTTP 내용 정리

### 주요 용어 한 줄 정리
- **HTTP**: 클라이언트와 웹 서버가 통신할때 시스템간에 원활한 소통을 위해 정해진 문서 전송 규칙
- **HTTP 요청 메서드**: 클라이언트가 서버에게 데이터를 조회, 추가, 수정, 삭제 등 어떤 행위를 하고 싶은 지 알려주기 위해 사용하는 명령어
- **HTTP URL**: 웹상에서 특정 데이터나 페이지의 위치를 나타내는 주소
- **HTTP 경로 변수와 쿼리 스트링**: URL 뒤에 구체적인 정보를 표기함
  - 경로 변수: url 경로 내에 포함됨. 특정 데이터를 식별하기 위해 사용하는 고유 식별값(숫자 or 문자)
  - 쿼리 스트링: url 맨 뒤 ? 뒤에 키-값 형태로 붙음. 웹페이지에 대한 추가정보 표기
**JSON**: 파이썬의 딕셔너리와 구조가 유사한 키-값 구조를 가짐. 자바스크립트의 객체 형태를 따르며, 시스템 간에 주고받기 편리하도록 텍스트 형태로 표현한 데이터 형식
**FastAPI**: 파이썬 기반으로 웹을 만들기 위해 만들어진 웹 프레임워크
**REST API**: 기존의 복잡한 방식 대신 HTTP를 활용해서 데이터와 HTTP 메서드를 매칭시켜서 데이터를 주고받을때 사용되는 세부적인 가이드라인

### HTTP 메서드
- GET: 글 목록 조회, 글 읽기
- POST: 글 쓰기, 댓글 쓰기
- PUT(PATCH): 글 수정, 댓글 수정
- DELETE: 글 삭제, 댓글 삭제

### API 명세서
API 명세서에 들어가야 할 4가지 요소
1. Method(메서드): GET, POST, PUT, DELETE 중 무엇인지
2. Endpoint(URL): 접속 주소는 무엇인지 (예: /posts)
3. Request(요청 데이터): 서버에 보내는 정보가 무엇인지 (예: title, content, password 등)
4. Response(응답 데이터): 서버가 결과를 잘 처리했는지, 어떤 데이터를 돌려주는지 (에: {"status": "success", "post_id": 1})
